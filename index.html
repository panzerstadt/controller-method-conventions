MVC's C

some conventions found in many modern MVC frameworks, like Laravel, AdonisJS, Ruby on Rails

Methods

index() "PhotoController.index"

GET

usually a GET request to the main endpoint, e.g. "/photos". This is the "R" of your CRUD

create() "PhotoController.create"

GET

the endpoint doesn't create, but is used to point to a page with a form to create a new "thing" e.g. "/photos/create" meaning something like "add new photo page"

store() "PhotoController.store"

POST

the "C" of CRUD. endpoint to save a user submitted form to the database

show() "PhotoController.show"

GET

show page by id, usually has a wildcard to match an id, e.g. "/photos/:id" --> "/photos/50" shows photo number 50

edit() "PhotoController.edit"

GET

again, not this endpoint doesn't do the editing, but rather shows a form to edit stuff e.g. "/photos/:id/edit" --> "/photos/36/edit" means show me the edit form for photo number 36 (cause i wanna add a description or tags)

update() "PhotoController.update"

PUT/PATCH

the "U" of CRUD. endpoint to update details for a given user provided id (usually through the UI on the "/edit" route)

destroy() "PhotoController.destroy"

DELETE

the "D" of CRUD. endpoint to delete a datapoint by the given id (which usually corresponds to a row in your database)
